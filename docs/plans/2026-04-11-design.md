# EhitusRadar — Spring Gardening Price Tracker

> Design document — 2026-04-11
> Status: APPROVED (pending implementation plan)

## Problem

No tool exists to compare prices of the same home improvement product across Estonian chains. Users manually visit Bauhof, Espak, Decora, Ehituse ABC to find the cheapest price for common products like Biolan muld 60L, Fiskars garden tools, Tikkurila paint.

## Target User

Home gardener doing spring shopping — soils, seeds, fertilizers, paints, garden tools. NOT professional builders buying bulk concrete. Emotional, seasonal, price-sensitive purchases.

## Scope: Spring Gardening Focus

### Target Categories
1. Mullad ja turbad — soils, peat, substrates
2. Väetised — fertilizers (Baltic Agro, Biolan, Substral, Compo)
3. Seemned — seeds (Baltic Agro, Horticom, Seston)
4. Värvid — paints (Eskaro, Vivacolor, Tikkurila, Sadolin)
5. Aiatööriistad — garden tools (Fiskars, Gardena, Makita, Bosch)
6. Lillepotid ja kastekannud — pots, watering cans (Prosperplast, Elho)
7. Muruniidukid ja trimmerid — lawn mowers & trimmers
8. Taimekaitsevahendid — plant protection

### Key Brands (appear across 3+ chains)
Biolan, Baltic Agro, Kekkilä, Grass, Horticom, Fiskars, Gardena, Makita, Bosch, Ryobi, Eskaro, Vivacolor, Tikkurila, Sadolin, Prosperplast

| Espak | WooCommerce + Algolia | POST `UTKUHA7JT0-dsn.algolia.net` index `vwwm_posts_product` | `b9c55a35ad2799389ce3d4584a5f9def` | 554 Fiskars, 21K+ products, **has EAN codes** |

### Tier 2: SSR Extraction

| Chain | Platform | Approach |
|-------|----------|---------|
| Bauhof | Vue/Nuxt + StorefrontUI | Extract `window.__NUXT__` from `/et/search?term=` pages. Names + SKUs confirmed. Prices need product page fetch or __NUXT__ variable decoder |

### Not in scope (for now)
- K-Rauta: LupaSearch API returns 401, category pages 403
- Bauhaus: Persistent 429 rate limiting + CAPTCHA
- DEPO: Minimal search, poor product data
- Homer: Magento HTML scraping possible but lower priority

## Golden Products Identification (Hybrid Bootstrap)

### Phase 1 — Category Seed
8 spring gardening categories (above). ~300 products per chain = ~1200 raw entries.

### Phase 2 — Cross-Chain Search
For each category, query all 4 chain APIs in parallel. Collect all products.

### Phase 3 — AI Normalization
Use Claude Haiku to normalize each product name:
```
Input:  "AIAMAA MUST MULD BIOLAN 60L"
Output: { brand: "Biolan", type: "aiamaa muld", variant: "must", size: "60L" }
```
Matching key: `brand|type|size` — products with same key across chains = golden.

### Phase 4 — Golden List
Products appearing in 2+ chains = tracked. Rank by:
1. Chain presence count (4/4 > 3/4 > 2/4)
2. Price variance (bigger spread = more interesting)
3. Category importance (soils > niche tools)

Target: 100-500 golden products.

### Proof of Concept Results
From manual matching of soils category alone:
- Biolan aiamaa must muld 60L: Decora €3.29, Ehituse ABC €3.20, Bauhof TBD
- 6 products confirmed in 3+ chains
- 14 products confirmed in 2+ chains
- Fiskars: 115-554 products per chain, many overlapping

## Architecture

### Stack (proven from HindRadar/PriceHNTR)

| Layer | Tech |
|-------|------|
| Framework | Next.js (Vercel) |
| Language | TypeScript |
| Styling | Tailwind CSS |
| Cache | Upstash Redis (6h TTL) |
| Database | Neon (golden products list + price history) |
| Hosting | Vercel |

### Data Model

```typescript
type ChainId = "decora" | "ehituse-abc" | "espak" | "bauhof";

interface Product {
  chain: ChainId;
  name: string;
  sku: string;
  ean: string | null;      // Espak has these
  brand: string;
  category: string;
  regularPrice: number;
  salePrice: number | null;
  inStock: boolean;
  imageUrl: string | null;
  productUrl: string;
}

interface GoldenProduct {
  id: string;
  normalizedName: string;  // AI-generated canonical name
  brand: string;
  type: string;            // product type
  size: string;            // 60L, 4KG, etc.
  category: string;        // mullad, väetised, etc.
  chains: ChainMatch[];    // one per chain
}

interface ChainMatch {
  chain: ChainId;
  sku: string;
  currentPrice: number;
  salePrice: number | null;
  lastChecked: string;     // ISO datetime
  priceHistory: PricePoint[];
}

interface PricePoint {
  date: string;
  regularPrice: number;
  salePrice: number | null;
}
```

### Cache Strategy (from HindRadar)
1. Check Redis cache (6h TTL)
2. Cache miss → fetch from chain API
3. Chain API down → serve stale cache
4. Redis down → direct API call

### Scraper Architecture
```
src/lib/
  scrapers/
    klevu.ts        — shared Klevu v2 adapter (Decora + Ehituse ABC)
    algolia.ts      — Espak Algolia adapter
    bauhof.ts       — Bauhof __NUXT__ extractor
  cache.ts          — Upstash Redis (same pattern as HindRadar)
  types.ts          — Product, GoldenProduct, ChainMatch
  matcher.ts        — AI normalization + cross-chain matching
  constants.ts      — chain configs, category definitions
```

### Klevu Adapter (shared by 2 chains)
Single function, parameterized by host + API key:
```typescript
async function searchKlevu(config: KlevuConfig, query: string): Promise<Product[]>
```
This is a major win — one adapter covers Decora + Ehituse ABC.

## Gotchas (from investigation)

| Issue | Solution |
|-------|----------|
| Klevu v2 needs POST with `context.apiKeys` | Not GET params — must send JSON body |
| Espak Algolia index is `vwwm_posts_product` not `wp_posts_product` | From WP plugin prefix |
| Bauhof __NUXT__ prices are minified variables | Extract names/SKUs from SSR, fetch prices via product pages |
| Product names vary wildly across chains | AI normalization via Claude Haiku |
| Ehituse ABC Klevu has no sale prices in listing | All show price=salePrice, discounts in product pages only |
| Espak has 21K+ products | Filter by gardening categories, don't index everything |
| Bauhof search returns max 32 results per page | Paginate with offset |
